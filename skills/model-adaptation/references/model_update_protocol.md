# Model Update Protocol

Use this when adding or revising model profiles.

1. Record date, source, and surface: web app, API, partner platform, or local model.
2. Separate official docs from community reports.
3. Update modality, parameters, reference support, negative support, duration, aspect ratio, resolution, seed/control, text/logo behavior, audio behavior, and failure modes.
4. Mark confidence: high for official/current docs or repeated project evidence, medium for strong but indirect guides, low for comparison mentions, unknown for missing evidence.
5. Add or update one example export and one test case.
6. Do not delete older caveats unless the new source directly supersedes them.
