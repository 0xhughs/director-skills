# Iteration Diagnostics

## Failure categories
- Ambiguous subject or action.
- Contradictory style, lighting, or camera directions.
- Prompt overload.
- Temporal overload for video duration.
- Reference asset conflict.
- Missing continuity anchors.
- Model unsupported capability.
- Safety, IP, or policy rejection.

## One-variable rule
If the output is mostly right, change one high-leverage variable at a time: subject wording, camera move, light source, reference image, negative constraint, aspect ratio, or duration.

## When to abandon
Start over or redesign when the failure is structural: impossible duration, central hand/mirror/text task in a weak model, too many characters, or unsupported feature dependency.
