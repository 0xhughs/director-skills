# Mastering gpt-image-2: an exhaustive prompt and craft manual

OpenAI **shipped GPT Image 2 (`gpt-image-2`) on April 21, 2026** as the underlying model for "ChatGPT Images 2.0," and this report focuses entirely on that model rather than any GPT Image 1 variant. The release matters because it is the **first OpenAI image model with native O-series "thinking" reasoning**, with end-to-end planning, web search and self-verification baked in, and because it took the **#1 position on every Image Arena leaderboard within 12 hours of launch with a +242 ELO lead** over Google's Nano Banana 2 — the largest single-model gap ever recorded there. GPT Image 2 reframes image generation from "rendering a description" to "executing a brief": near-perfect text and multilingual rendering, surgical multi-turn editing, up to eight coherent images per call, and resolutions to 4K. Predecessors in the lineage — `gpt-image-1` (April 2025), `gpt-image-1-mini` (October 2025), `gpt-image-1.5` (December 2025), and DALL·E 2/3 (retired May 12, 2026) — remain useful only for niches such as transparent backgrounds. The single snapshot in production is **`gpt-image-2-2026-04-21`**, with a December 2025 knowledge cutoff and an `auto`-rolling alias `chatgpt-image-latest`. Everything that follows assumes this model. Throughout, claims are tagged as **OFFICIAL** (OpenAI / Microsoft documentation), **COMMUNITY-VERIFIED** (multiple independent practitioners), or **SPECULATIVE**.

## What is actually different about this model

OpenAI publicly describes GPT Image 2 as a "generalist model" or "GPT for images" and **declined in launch press briefings to confirm whether it is autoregressive or diffusion-based** (TechCrunch, VentureBeat). The image-generation guide hints autoregressively at the architecture by stating images are produced as "specialized image tokens" whose count drives latency and cost, but no first-party document is explicit. Practically, the model behaves like an LLM with image output: it rewards natural-language briefs, follows complex multi-part instructions, recovers from negative phrasing ("no watermark"), and **does not respect Stable-Diffusion-style weight syntax such as `(word:1.3)`** — there is no documented evidence that such weights affect output, and OpenAI's own cookbook explicitly steers users toward "skimmable templates over clever prompt syntax."

The headline architectural addition is **"Thinking mode."** When a thinking-class model is selected in ChatGPT, or when the `image_generation` tool is called inside the Responses API by a reasoning-capable mainline model such as `gpt-5.4` or `gpt-5.5` with `reasoning: {"effort": "high"}`, the system **plans the layout, can invoke web search, generates multiple distinct candidates, and self-checks output before returning**. OpenAI demonstrated up to **eight coherent images from a single prompt** with shared characters, palettes and props — the core production primitive for storyboards, multi-format campaigns and outfit grids. There is no documented top-level `thinking` flag on the bare Image API; community claims of a `quality="thinking"` value are unverified and contradicted by official docs that list `quality` only as `low | medium | high | auto`. **Thinking mode is gated to Plus, Pro, Business and Enterprise subscribers in ChatGPT**; free users get Instant mode only.

Compared with `gpt-image-1.5`, the practical wins are: **non-Latin text rendering at ~99% character accuracy** in Japanese, Korean, Chinese, Hindi and Bengali (officially highlighted) plus working Devanagari, Cyrillic, Greek and Arabic; **arbitrary resolutions** (each edge a multiple of 16, max edge < 3,840 px, ratio ≤ 3:1, 0.66–8.29 MP total — with anything above 2,560×1,440 marked **experimental**); **up to 16 reference images** with locked high-fidelity processing; **far better instruction following on dense layouts** (infographics, UI mockups, menus); and a **fix for the "yellow tint" color cast** that plagued v1.5. Two losses are notable: **transparent backgrounds are not supported** (`background: "transparent"` is rejected — fall back to `gpt-image-1.5` for cutouts), and the `input_fidelity` parameter is **disabled** because every input is now processed at high fidelity, which makes reference-heavy edits more expensive than the same-resolution generation.

Against the broader 2026 field, GPT Image 2's edge is concentrated where text, layout and reasoning intersect. **Nano Banana Pro (Gemini 3 Pro Image)** still beats it on portrait skin texture, group shots with five or more people, multi-reference fusion at 14 images, and 4K cost ($0.24 vs ~$0.41 per image via fal.ai). **Imagen 4 Ultra** is the photorealism leader at $0.06 per image but has no native edit endpoint and trails on text. **Midjourney V8.1** ships fast and atmospheric but **has no public API** and loses on text rendering. **FLUX.2 \[pro\]** and **\[klein\]** (Black Forest Labs, late 2025/early 2026) are the open-weight ceiling at ~$0.030/MP with strong typography. **Ideogram V3** remains best-in-class for poster typography at ~90% spelling accuracy, and **Recraft V3** is the only major model with native SVG vector output. **Adobe Firefly Image 4** holds the "commercially safe" position via licensed training data and now actually bundles GPT Image 2, Nano Banana Pro and Runway Gen-4 inside the Firefly app. The "what to use" decision therefore rarely collapses to a single winner — it depends on whether your bottleneck is text fidelity (GPT Image 2), portrait realism (Nano Banana Pro / Imagen 4), vector output (Recraft), or licensing safety (Firefly).

## The parameter surface, in full

GPT Image 2 is exposed across `v1/images/generations`, `v1/images/edits`, `v1/responses` (via the `image_generation` tool), `v1/chat/completions`, plus listings for `v1/realtime`, `v1/assistants` and `v1/batch`. The model card explicitly marks **streaming, function calling, structured outputs, fine-tuning, distillation and predicted outputs as not supported**, though `partial_images` previews are supported on the Image and Responses endpoints. **Use of GPT Image models requires API Organization Verification.** The full parameter table below consolidates OpenAI, Microsoft Foundry and the OpenAI Cookbook.

| Parameter | Allowed values | Behavior on gpt-image-2 |
|---|---|---|
| `model` | `gpt-image-2` (alias) / `gpt-image-2-2026-04-21` (snapshot) / `chatgpt-image-latest` (rolling) | Pin the snapshot in production; alias rolls forward silently |
| `prompt` | string, max 32,000 characters | Long descriptive briefs outperform short prompts; structured/labeled sections beat single paragraphs for complex briefs |
| `quality` | `low` / `medium` / `high` / `auto` | Default `auto`; `low` is the latency-optimized tier; small text needs `medium` or `high` |
| `size` | any WIDTH×HEIGHT meeting constraints | Both edges multiples of 16; max edge < 3,840 px; ratio ≤ 3:1; 655,360 ≤ pixels ≤ 8,294,400; >2,560×1,440 is **experimental**; oversize requests auto-resize down |
| `n` | 1–10 (OpenAI / Replicate); thinking-mode batches up to 8 with cross-frame consistency | `n>1` multiplies cost linearly |
| `background` | `auto` / `opaque` (`transparent` rejected on v2) | Use `gpt-image-1.5` for transparent PNGs |
| `output_format` | `png` (default) / `jpeg` / `webp` | Azure does not support `webp`; image API always returns base64 (`b64_json`) |

| `moderation` | `auto` (default) / `low` | `low` only relaxes age-appropriateness; does not unblock NSFW, minors, hate, deepfakes, IP, or named celebrities |
| `input_fidelity` | locked / not user-settable | Every input processed at high fidelity, raising input-token cost on edits |
| `partial_images` | 0–3 | Each preview adds ~100 image output tokens |
| Mask (edits) | PNG with alpha channel, same dims as input, < 50 MB | Transparent areas regenerate; opaque areas preserve; masking is **prompt-guided not pixel-strict** ("the model uses the mask as guidance, but may not follow its exact shape with complete precision") |
| Seed / determinism | not exposed | No first-party reproducibility control documented |

Token-based pricing is **$8/M image input, $2/M cached image input, $30/M image output, $5/M text input, $1.25/M cached text input, $10/M text output** — Microsoft notes "no billing for output text tokens" on Azure. The effective per-image costs at 1024×1024 land at **~$0.006 (low), ~$0.053 (medium), ~$0.211 (high)**; portrait/landscape 1024×1536 and 1536×1024 are slightly cheaper because of token-tile geometry; 2K runs about **$0.28** and 4K via fal.ai about **$0.41**. **Every reference image in an edit is billed at high fidelity**, so a generate-then-four-edits workflow can cost two to three times the headline number — caching repeated inputs at the cached-input rate ($2/M) is the largest single cost lever and recovers ~75% of that overhead. Rate limits scale by tier from **5 images/min (Tier 1)** to **250 images/min (Tier 5)**, with Tier 5 requiring $1,000 of spend and a 30-day-old account; cold-path Tier-1 traffic throttles aggressively, so retry budgets need exponential backoff with jitter.

## Prompt anatomy that the model actually rewards

OpenAI's Cookbook gives an unusually direct rule on structure: **write prompts in the order background/scene → subject → key details → constraints, plus the intended use ("ad," "UI mock," "infographic")** to set the model's mode and level of polish. For complex briefs, **labeled, line-broken sections beat one long paragraph**; for production systems, prefer a "skimmable template" over clever syntax. The community has converged on a five-slot scaffold (popularized by fal.ai's prompting guide and Atlabs) that maps cleanly onto the Cookbook ordering:

```
Scene: where, time of day, environment
Subject: who or what is the focus
Important details: materials, clothing, texture, lighting, camera, lens, composition, mood
Use case: editorial photo / UI mock / poster / infographic / concept frame
Constraints: no watermark, no extra text, preserve face, preserve layout
```

JSON-style prompt bodies, descriptive paragraphs, instruction-style and tag-based formats all work; the constraint is clarity, not syntax. **Prompt length pays off when it adds visual specificity, not adjectives**: replacing "stunning, 8K, masterpiece, cinematic" with concrete physical facts ("overcast daylight, 50 mm, brushed aluminum, visible pores, slight film grain") is the single highest-leverage change moving from DALL·E 3 / Midjourney habits to GPT Image 2. Legacy "magic words" (8K, masterpiece, trending on ArtStation, intricate details) **contribute nothing** on this model and are explicitly called out as 2023-Midjourney noise. The one praise-word that does pull weight is **"photorealistic,"** which OpenAI confirms engages the model's photorealism mode.

Negative phrasing is handled in natural language. The Cookbook recommends **stating exclusions and invariants as plain sentences**: "no watermark," "no extra text," "no logos/trademarks," "preserve identity/geometry/layout/brand elements." There is no Stable-Diffusion-style negative-prompt field, and **weighted negatives like `no noise:1.2` are not supported by any OpenAI-published behavior** (treat as speculative). Positional/token weighting in the SD sense is similarly unsupported; emphasis is best produced by repetition, by ALL CAPS for literal text strings, and by explicit constraint sentences. For edits, the canonical pattern is **"change only X" plus "keep everything else the same," with the preserve list repeated on every iteration** to suppress drift; if the edit must be surgical, also forbid changes to saturation, contrast, layout, arrows, labels, camera angle and surrounding objects.

## The craft layer: what produces predictable, professional output

**Subjects** should be specified at scale-and-interaction granularity, not just identity. The Cookbook's working examples — "full body visible, feet included," "child-sized relative to the table," "looking down at the open book, not at the camera," "hands naturally gripping the handlebars" — outperform demographic adjectives because the model interprets them as composition constraints. **Shot vocabulary that is officially documented and verified working** includes close-up, medium, wide, top-down, eye-level, low-angle, plus community-verified Dutch angle, three-quarter, POV and overhead. Spatial direction language ("logo top-right," "subject centered with negative space on left," "rule of thirds with leading line from foreground rocks to lighthouse," "S-curve composition") is respected reliably.

**Lighting terminology** that produced expected results across multiple independent practitioner tests includes golden hour, blue hour, overcast daylight, soft diffuse, hard light, softbox, high-key, low-key, high-contrast, rim light, sidelight from camera left, backlight, key/fill/back, and "practical light sources only." Cinematographer references work as *look* shorthand — **"Roger Deakins-inspired color grading with teal shadows and warm skin tones"** is a documented working pattern from independent testers (James Palm, Atlabs). Older cinematic terms (Rembrandt, chiaroscuro) are not specifically validated in the Cookbook but are consistent with the model's strong art-history knowledge and behave as expected in practice. **Hex codes are functional in production prompts** — OpenAI's own pitch-deck example specifies a "color palette with hex codes," and Atlabs' brand-guideline template treats hex specification as core. Named grading recipes (teal-and-orange, bleach bypass) work; ENR, Kodachrome and similar are anecdotally supported but uncited.

**Material vocabulary** is one of the strongest levers. The Cookbook prescribes being "concrete about materials, shapes, textures" and adds **"targeted quality levers only when needed (e.g., film grain, textured brushstrokes, macro detail)."** Verified working terms include visible pores, sun texture, fabric weave, brushed aluminum, chipped paint, wet concrete, contact shadows, slight film grain, and natural color balance. Subsurface scattering and microsurface detail appear in community libraries but lack first-party validation. **Camera and lens specs are interpreted "loosely"** per the Cookbook — they work as aesthetic shorthand for bokeh, lens compression and grain, not as physical simulators. Focal lengths (35 mm, 50 mm, 85 mm), apertures (f/1.4, f/2.0), formats ("anamorphic 2.39:1," "35 mm film photograph"), and even named bodies/lenses ("ARRI ALEXA with Cooke S5 lenses") successfully invoke the corresponding *look*; film stocks like Portra 400 and Cinestill 800T behave similarly. Sensor types do nothing observable beyond mood-cue.

**Style invocation** has tightened since the GPT Image 1 Studio Ghibli viral moment of March 2025. The current safety stack carries forward an explicit **refusal for prompts requesting the style of living artists** — Hayao Miyazaki, Makoto Shinkai, Banksy, Takashi Murakami all block. **Studio names ("Ghibli style"), art movements (Bauhaus, Art Deco, Pop Art, Ukiyo-e), deceased artists (Van Gogh, Monet, Hokusai, Basquiat, Warhol), and named cinematographers** as look-references continue to work. Style transfer best practice is unchanged: name what stays (palette, brushwork, grain, edges) and what changes (subject, scene), plus hard "no extra elements" guards.

## Text rendering — the headline capability, used correctly

GPT Image 2's text accuracy is variously reported at 95% (PixVerse hands-on, ~19/20 first-attempt legibility), 95–99% (OpenAI/community average), and >99% on benchmark cards. The discipline that delivers it is straightforward and officially documented: **wrap literal text in quotes or ALL CAPS, specify typography (font style, weight, size, color, placement), spell tricky words letter-by-letter, and use medium or high quality for small or dense text.** The community-converged pattern is `EXACT TEXT: "..."` followed by `Render the text verbatim. No extra words. No duplicate text.` For long copy, cap each rendered string at five words or fewer where possible and split into multiple lines with explicit line-by-line specification. Multilingual signage works best when you **paste literal native characters** rather than asking the model to translate during generation — Segmind's verified single-pass multilingual pattern places `'मुंबई पुस्तक भंडार'` directly in quotes as line one, `'Mumbai Book Store'` as line two, with explicit ordering.

UI mockups are a standalone use-case that the Cookbook treats as its own discipline: **"describe the product as if it already exists"** — focus on layout, hierarchy, spacing, real interface elements, and avoid concept-art language. Include status-bar time ("9:41 AM"), exact bullet copy, an accent color, and constraints like "soft card shadows, perfect legibility, generous spacing" plus "no watermark, no real app branding." For pitch-deck-style slides, the Cookbook's worked example uses Markdown emphasis inside the prompt (`**TAM:** $42B`) to convey hierarchy and pairs typography ("modern sans-serif typography like Inter") with layout discipline ("avoid clip art, stock photography, decorative elements"). For QR codes and barcodes — which GPT Image 2 actually generates as scannable artifacts ~60–70% of the time — supply the encoded data, demand "high contrast, no JPEG fringing, clean vertical bars," set `quality: high`, and validate the output with a real scanner before shipping. Logos remain the biggest text-adjacent weakness: **brand-logo reproduction is unreliable across iterations**, so production work composites real logos in post.

## Advanced workflows: thinking mode, editing, consistency

The decision rule for **Thinking mode** (Plus/Pro tier in ChatGPT, or Responses-API reasoning model on the API) is "use it when the layout matters, skip it when speed matters." Concretely, invoke Thinking when the prompt exceeds about 30 words, contains structured information (text, layout, multilingual), needs web-grounded data, or requires multi-image continuity (`n > 1`); skip it for fast iteration, batch backgrounds, and simple subjects. Latency rises from roughly 3 seconds (Instant) to 15–40 seconds typical, with complex eight-image batches occasionally reaching two minutes. Web search activates only inside Thinking mode and pulls real-time facts before rendering — useful for current sports records, just-released products, or fact-grounded infographics. Self-verification means the model reviews its own draft, particularly for text accuracy and instruction adherence, and may regenerate before returning. Eight-image batches are how you get the consistent multi-panel comic, the multi-aspect campaign in one call, or the eight-outfit reference grid OpenAI used in its launch demo.

**Editing** runs through `POST /v1/images/edits` (multipart) or via the Responses API's `image_generation` tool. The cookbook prescribes labeling each input image by **index and role** ("Image 1: product photo. Image 2: style reference."), then describing the interaction — "apply Image 2's style to Image 1," "place the bird from Image 1 onto the elephant in Image 2." Masks are PNGs of identical dimensions to the input where transparent regions regenerate and opaque regions preserve, but OpenAI explicitly notes the **mask is prompt-guided rather than pixel-strict**. Surgical multi-turn editing without drift comes from disciplined preserve-lists: every iteration restates "Same green hooded tunic, same facial features, proportions, color palette, same gentle personality" alongside the change list; every iteration also bans drift in saturation, contrast, layout, arrows, labels, camera angle and surrounding objects. The Responses API adds genuine multi-turn memory via `previous_response_id`, enabling conversational editing without manual chaining.

**Character and style consistency** follow a documented two-step "anchor" pattern. Generate the character once on a plain background to lock identity. Then use `images.edit` with that anchor as input and a new scene prompt; explicitly enumerate the consistency constraints on every continuation. For storyboards beyond eight panels, the EvoLink-style pattern of a single 3×4 grid image with scene numbers and 1–2 line descriptions is more reliable than chaining many separate generations — the model plans the global composition once. **World-knowledge exploitation** — leveraging real brands, locations, scientific concepts and historical scenes — works strongly on this model (Cookbook example: "Bethel, New York on August 16, 1969" infers Woodstock without naming it), but the December 2025 cutoff means anything more recent requires Thinking mode + web search. **Iterative workflows** scale by chaining a reasoning model (`gpt-5.4` or `gpt-5.5` in the Responses API) that authors structured prompts and calls the `image_generation` tool — an emerging "image generation as front-end for coding agents" pattern (Latent Space) where Codex generates a UI mockup as image and then implements against it. `partial_images` (0–3 progressive previews, +100 image output tokens each) supports interactive UX; skip it for batch.

## Failure modes, debugging, and the content-policy edge

The officially-acknowledged limitations are physical-world reasoning (origami, gears, optical reflections, Rubik's cubes can be visually plausible but physically wrong), repetitive fine detail (Simon Willison's Where's-Waldo raccoon test failed at default settings and only succeeded at `quality: high` + 3,840×2,160 for ~$0.40 per attempt), iterative-edit diminishing returns after a few rounds, and counting/duplication errors when asked for a specific number of people. Community-verified failure patterns and their fixes form a coherent taxonomy: **subject drift** comes from missing or unrepeated preserve-lists; **style bleed** from unlabeled reference inputs; **anatomy errors** from omitted lens/light specs that drop the model into a generic stylized mode; **text misspellings** from underspecified placement or non-quoted strings; **composition collapse** from single-paragraph dense prompts that should be broken into labeled sections; **perspective errors** in sketch-to-image jobs from missing anchors ("preserve the exact layout, horizon line, river path, mountain placement"). A known **noise-amplification bug** in the ChatGPT UI causes pattern-amplified or near-identical outputs across regenerations within the same session — the documented workaround is to reload the web page between generations.

The diagnostic process that experienced practitioners run on a failing prompt is a binary search at low quality (~$0.006 per attempt): split the prompt in half, retry, repeat to localize the trigger. For moderation failures, distinguish "rejected" at the input filter (Stage 1, fixable by rewriting) from "filtered" at the output filter (Stage 2, the entire scene needs redesign) — pre-checking with `omni-moderation-latest` (free) before paying for generation is standard. **The decision between iterating and abandoning** turns on whether the first pass is structurally correct: if composition, aspect ratio, or fundamental subject is wrong, restart from scratch; if the structure is right and only cosmetics (lighting tone, one element, color cast) are off, iterate with small change-preserve edits. Three failed edits without convergence is the practical abandon-trigger.

The content-policy architecture is two-stage — an upstream text classifier that scans the prompt with about a 22% block rate on adversarial inputs in instant mode, plus a downstream multimodal safety reasoning model that checks both input images and the final generated image. **Edit-endpoint filtering is stricter than generation-endpoint filtering** by design, to suppress deepfakes and clothing-removal. The categories that consistently refuse are real-person portraits and named celebrities (tightened after the October 2025 Bryan Cranston incident), living-artist style references, copyrighted characters and IP (Disney, Marvel, Pokémon, Studio Ghibli's named characters even though "Ghibli style" itself remains permitted), realistic violence and gore, sexual suggestion or revealing-clothing adjectives ("tight," "revealing," "sexy," "bikini" all trigger), realistic minors, and hate symbols. The **`moderation: "low"` parameter relaxes only age-appropriateness** — it does not unblock any of the seven hard categories. Legitimate rephrasing strategies that preserve creative intent: replace artist names with movements or studios; route living-artist intent to a deceased equivalent; abstract IP characters into descriptive features ("round yellow electric cartoon creature with red cheeks and pointed ears"); use a text-LLM pre-pass with a system prompt that strips real names, brands and IP; substitute emotional/atmospheric language for trigger words ("bloody" → "deep red tones, dramatic"); and, if the edit endpoint blocks, downgrade to a description-based generation. Every output is tagged with **C2PA content credentials** and an imperceptible watermark, though OpenAI's product lead has acknowledged that metadata strips on screenshot or recompression and that Twitter/X's CDN systematically destroys C2PA on upload (per arXiv 2604.25370).

## Production-ready prompt templates by use case

The following use-case templates are either drawn from the OpenAI Cookbook or from independent practitioners (fal.ai, Atlabs, Segmind, James Palm) who have tested and published them on GPT Image 2. Each is calibrated to the model's strengths and known failure modes.

**Photorealistic editorial portrait** (Cookbook §4.3 pattern; 1024×1536, `quality: medium` or `high` for skin closeups):
```
Create a photorealistic candid photograph of [subject].
[Skin/material details: visible wrinkles, pores, sun texture].
Shot like a 35mm film photograph, medium close-up at eye level, using a 50mm lens.
Soft coastal daylight, shallow depth of field, subtle film grain, natural color balance.
The image should feel honest and unposed, with real skin texture and everyday detail.
No glamorization, no heavy retouching.
```

**Product hero with brand integrity** (Cookbook §5.4; 1024×1024 or 3:2, `quality: high`):
```
Extract the product from the input image and place it on a plain white opaque background.
Output: centered product, crisp silhouette, no halos/fringing.
Preserve product geometry and label legibility exactly.
Add only light polishing and a subtle realistic contact shadow.
Do not restyle the product; only remove background and lightly polish.
```

**UI mockup with literal copy** (Cookbook §4.8; 1024×1536 mobile, `quality: high`):
```
Create a clean mobile app screenshot for a minimalist to-do app called DAYBREAK.
Top status bar reads 9:41 AM. Title: DAYBREAK. Subtitle: Tuesday, 23 April.
Four tasks listed:
- Review quarterly notes
- Call mom
- Ship the image update
- Pick up bread
One task is checked off.
Muted cream background, deep navy accent, rounded sans serif, soft card shadows,
perfect legibility, generous spacing. Place the UI in an iPhone frame.
No watermark. No real app branding.
```

**Multilingual signage** (Segmind verified single-pass; 1536×1024, `quality: high`):
```
A photorealistic cinematic shot of [scene] at [time].
In the foreground a chalkboard A-frame easel reads in clean legible chalk:
first line 'मुंबई पुस्तक भंडार',
second line 'Mumbai Book Store',
smaller third line 'Open Daily 9 am to 9 pm'.
Shallow depth of field, 35mm f/2.0, film grain.
Render text verbatim. No extra words. No duplicate text.
```

**Logo / brand-identity exploration** (Cookbook §4.5; 1024×1536, `n: 4`):
```
Create an original, non-infringing logo for a company called Field & Flour, a local bakery.
The logo should feel warm, simple, and timeless. Clean vector-like shapes,
strong silhouette, balanced negative space.
Favor simplicity over detail so it reads at small and large sizes.
Flat design, minimal strokes, no gradients unless essential.
Plain background. Single centered logo with generous padding. No watermark.
```

**Children's-book character anchor + continuation** (Cookbook §6.4): generate the character on a plain background first, then use `images.edit` with that anchor as input plus the scene continuation prompt, repeating identity invariants every iteration ("same green hooded tunic, same facial features, proportions, color palette, same gentle personality, same watercolor look — do not redesign the character").

**Pitch-deck slide** (Cookbook §4.10; 1536×864, `quality: high`):
```
Create one pitch-deck slide titled "Market Opportunity".
TAM/SAM/SOM concentric-circle diagram in muted blues and grays.
Specific numbers: TAM: $42B, SAM: $8.7B, SOM: $340M.
Bar chart 2021–2026 showing subtle upward trend.
Footnotes: "AGI Research, 2024" and "Internal analysis".
Modern sans-serif typography like Inter, polished spacing.
Avoid clip art, stock photography, gradients, shadows, decorative elements.
```

**Multi-panel comic / storyboard** (Cookbook §4.7; Thinking mode, `n: 4–8`): define each panel as a concrete visual beat — concrete actions and camera positions, not abstract emotion verbs which collapse into ambiguous panels — and demand consistent character, lighting and palette across panels. **Architectural object swap**: "In this room photo, replace ONLY the white dining chairs with chairs made of natural oak wood. Preserve camera angle, room lighting, floor shadows, and surrounding objects. Photorealistic contact shadows and believable wood grain." **Marketing creative with billboard text** uses the literal-text-in-quotes pattern with explicit typography ("Bold sans serif, centered, high contrast, clean kerning") and constraints ("no extra words, no duplicate text, no additional logos"). For **infographics**, set `quality: high` and use the structured-data pattern from the Cookbook's coffee-machine flow example or the labeled-grid pattern that scored decisively against Nano Banana Pro in the 3×3 clothing-grid head-to-head.

## Workflow integration and cost discipline

The decision among access surfaces is not subtle: the **API gives you full parameter control** (custom sizes, masks, snapshot pinning, response_format), **ChatGPT consumer is the cheapest path to Thinking mode** for low-volume use ($20/mo Plus beats rerolling Instant 4–5× for anyone making 20–200 images per month), **Codex turns image generation into a repo-aware design tool** (anecdotal iteration counts dropped from 47 to 6 on UI work in dev.to's documented case), and **Microsoft Foundry / Azure OpenAI** layers enterprise governance, regional residency and Azure AI Content Safety on top. Day-one platform integrations include Figma, Canva, Adobe Firefly Boards, fal.ai, Replicate, Hermes Agent and OpenRouter; fal.ai exposes both endpoints with mask uploads, queueing, webhooks, and BYO-key routing. Sora-side: the Sora team was reportedly restructured before this launch and the Sora product was retired April 26, 2026, per secondary reporting; image-to-video handoffs now route to **Veo 3.1** (cinematic, native audio sync), **Runway Gen-4.5** (best character consistency from a single ref, #1 on Video Arena), or **Kling 3.0** (~$0.50 per 4K clip — best per-clip cost).

Cost discipline starts with **pinning the snapshot ID `gpt-image-2-2026-04-21` in production** so the alias does not roll under a running batch. Three cost levers move the needle. First, **caching repeat reference images** drops their input billing from $8/M to $2/M tokens — a 75% saving, and the single largest practical lever in edit-heavy workflows. Second, **routing by complexity**: low quality at $0.006 per 1024-square is the binary-search debugging tier and the bulk-draft tier; medium at $0.053 is the production sweet spot for most marketing work; high at $0.211 is reserved for text-heavy and dense-layout finals. A 10K-image-per-month modeled scenario lands at roughly $2,110 on all-high gpt-image-2, $390 on Nano Banana Pro for portrait-heavy work, and ~$1,250 on a 50/50 hybrid where text-and-layout work goes to gpt-image-2 and pure photoreal portraits route to Nano Banana Pro. Third, **don't pay 4K premium when you don't need it** — gpt-image-2 4K via fal.ai is $0.41 versus Nano Banana Pro 4K at $0.24, and 2K + Magnific or Topaz upscaling typically beats both for creative hallucination of texture. Skip upscaling entirely when the native 4K output is acceptable; reach for Topaz Gigapixel for faithful enlargement and print, Magnific or Krea for creative re-texturing, Enhancor specifically to break the "plastic AI" look on portraits, and Upscayl (free, Real-ESRGAN, AGPL-3.0) for unlimited local work.

For prompt management at scale, the 2026 toolchain has consolidated around a small set of options. **Langfuse** (open source, MIT, OpenTelemetry-native) is the strongest default for granular observability and self-hosting; **PromptLayer** is the choice for engineering teams that need non-technical-stakeholder collaboration with visual prompt builders, A/B tests and Git-like rollback; **Helicone** is the lightweight proxy gateway for cost control, caching and model routing with one-line integration; **Braintrust** acts as a direct OpenAI proxy with no instrumentation. The recommended pattern is: store prompts outside code in a registry, tag them with feature, owner and approval state, require diffs and approvals before promoting to production, build a 100–200-case golden dataset and run regression evals on every prompt change, route all API calls through a proxy for caching and cost telemetry, and sample 1–5% of production runs for nightly eval.

## Conclusion: how this changes the work

Two shifts matter more than any individual technique in this report. First, GPT Image 2 collapses the historical gap between "AI-generated" and "production-ready" specifically for the artifacts where text and layout meet — UI mockups, infographics, posters, packaging, multilingual signage, multi-panel comics. The model is not a Midjourney replacement for atmospheric art; it is a **reliable execution layer for design briefs that previously required a designer**. Second, the introduction of native reasoning means that prompt engineering for images is starting to converge with prompt engineering for text agents: the model rewards structured briefs with explicit constraints, intended use, change-preserve discipline on edits, and identity invariants restated every turn — exactly the patterns that make reasoning-model agents reliable.

The hidden capability that separates amateur from professional output is not a secret parameter: it is **discipline at the brief level**. Replace praise adjectives with physical facts. Quote literal text and forbid duplicates. Label reference images by index and role. Repeat preserve-lists every iteration. Pin the snapshot ID. Cache repeated inputs. Use Thinking mode when layout matters and Instant when speed matters. Route portraits to Nano Banana Pro and text-heavy work to GPT Image 2. The model exposes far more capability than its parameter surface suggests — but only to operators who treat each generation as the execution of a structured plan rather than the rendering of a description. Three areas remain genuinely unsettled and should be watched closely over the next two quarters: official confirmation of the architecture (autoregressive vs diffusion), an exposed thinking/reasoning toggle on the bare Image API, and the eventual addition of seed-based reproducibility — none of which OpenAI has documented as of early May 2026.
